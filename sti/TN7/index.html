I- Rappel:

Les étapes de création d’un site Web :
  • Développement
  • Etude de faisabilité
  • Rédaction cahier des charges
  • Conception de la navigation
  • Réalisation technique
  • Hébergement
  • Référencement
  • Mise à jour E
  • Evaluation

II- Hébergement :

L’hébergement permet de mettre le site en ligne (relié en permanence à Internet). Pour cela il faut faire recours à un hébergeur, c'est-à-dire une société proposant l’hébergement (hosting) de sites web sur des serveurs spécialisés connectés en permanence à Internet. On distingue : • Hébergeurs gratuits : Proposent gratuitement un espace disque sur un serveur. Exemples : - www.fr.gd - www.hebergement-gratuit.com - www.ton-hebergement-gratuit.com • Hébergeurs professionnels : Ce type garantit un service de qualité et sécurité. Exemples : - www.elassar.com - www.webserveurs.com - www.punchost.com 

III- Référencement :

 Le référencement (Web positionning) a pour objectif de faire connaître le site aux différents moteurs et outils de recherche. Dans le référencement, on trouve : ? L’indexation : Faire connaître le site auprès des outils de recherche grâce aux formulaires que ceux-ci proposent. ? Le positionnement et le classement : consiste à positionner le site ou certaines pages du site en première page de résultat pour certains mots-clés. Il existe essentiellement deux méthodes de référencement : • Utilisation des services d’un "multi référenceur" spécialisé • Référencement manuel Pour être indexé correctement par les moteurs de recherches, il faut intégrer dans les codes html de chaque page Web des données (telles que le titre du site, sa description, une liste de mots-clés). Ces informations sont insérées dans les balises métas : <meta >. Exemples des sites de référencement gratuits : - http://www.ref-ici.com - http://www.refgratuit. - http:// myannuaire.com IV- Sécurisation : ? Plusieurs techniques et méthodes sont offertes pour sécuriser les données d’un site Web. Certaines sont réalisées par programmation (mot de passe, sessions, redirection…), d’autres à l’aide de fichiers de configuration des différents serveurs (Web, ftp, SGBD,..). ? La sécurisation peut toucher une page, un ensemble de page ou un ou plusieurs répertoires d’un site.
